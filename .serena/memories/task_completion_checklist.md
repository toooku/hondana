# タスク完了時のチェックリスト

## コード品質

- [ ] コードがプロジェクトのスタイル規約に従っている
- [ ] 型ヒントが適切に記述されている
- [ ] Docstringが適切に記述されている
- [ ] 命名規則が一貫している

## テスト

- [ ] 新しい機能に対するテストが追加されている
- [ ] 既存のテストがすべて通過している
  ```bash
  pytest tests/ -v
  ```
- [ ] テストカバレッジが適切に維持されている
  ```bash
  pytest tests/ --cov=src --cov-report=html
  ```

## ドキュメント

- [ ] README.mdが更新されている（必要に応じて）
- [ ] 新しい機能の使用方法がドキュメント化されている

## 動作確認

- [ ] 新しい機能が期待通りに動作する
- [ ] CLIコマンドが正常に動作する（該当する場合）
- [ ] Webインターフェースが正常に動作する（該当する場合）

## リファクタリング

- [ ] コードの重複がない
- [ ] 適切な抽象化が行われている
- [ ] エラーハンドリングが適切に実装されている

## Git

- [ ] 変更が適切にコミットされている
- [ ] コミットメッセージが明確で意味のあるものになっている

## 注意事項

- このプロジェクトには明示的なリンティングやフォーマットツール（black, flake8, mypyなど）の設定は見つかりませんでした
- テストはpytestを使用して実行します
- コードスタイルはPythonの標準的な規約に従っています
